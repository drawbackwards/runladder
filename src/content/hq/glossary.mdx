## Why this exists

Half the team drift on Ladder words shows up as inconsistent UI copy and bad sales conversations. This page is the single source of truth for what a word means inside Ladder.

If a word here conflicts with how we use it in code or copy, the code or copy is wrong, not the glossary. File a PR.

## The framework

| Term | Definition |
| --- | --- |
| **Ladder** | The framework. Also the product. Five rungs of quality, from Functional to Meaningful. |
| **Rung** | One of the five levels. Plural "rungs." Use this word, not "tier" (that's the price tier). |
| **Score** | A 1.00–5.00 decimal Ladder score. Two decimal places, always. |
| **Level** | The named rung a score lands on. Synonym of "rung" in user-facing copy, but "rung" preferred in framework discussion. |
| **Meaningful (5.00)** | Irreplaceable. The product is part of the user's life. |
| **Delightful (4.00–4.99)** | Anticipates needs. Users refer others. |
| **Comfortable (3.00–3.99)** | No thinking required. The modern minimum. |
| **Usable (2.00–2.99)** | Tasks complete with effort. |
| **Functional (1.00–1.99)** | The user fights the product. |
| **The modern minimum** | Phrase for 3.0 / Comfortable. Use when explaining the bar. |

Always list rungs 5 → 1. See [Brand](/hq/brand#rules-that-dont-bend).

## Surfaces

| Term | Definition |
| --- | --- |
| **Surface** | One of the five product touchpoints: web, Figma plugin, Claude Skill, Pulse, API + MCP. |
| **Web** | runladder.com. The web app + marketing site. |
| **Plugin** | Ladder for Figma. The Figma plugin. |
| **Skill** | Ladder for Claude. The Claude.ai skill. |
| **Pulse** | Ladder Pulse. Scoring from customer feedback data. |
| **API** | The public Ladder API. Includes the MCP server for AI agents. |

## Roles and tiers

See [Roles](/hq/roles) for the full matrix. Quick reference:

| Term | Definition |
| --- | --- |
| **Free** | Sign-up tier, 5 lifetime scores, base score only. |
| **Pro** | $1,000/mo self-serve on Stripe, individual, 2,000 scores/month, all surfaces. |
| **Team** | Agency engagement via MSA + SOW (Beta on /pricing). Publicly Custom. Capital T. |
| **Teams** | Plural noun ("Teams customers"), referring to multiple Team engagements collectively. |
| **Team Member** / **Team Leader** | Roles within a Teams engagement. |
| **Pulse User** | Customer on a Pulse engagement. |
| **Admin** | Internal Drawbackwards admin (Ward, Michael, Jordan). |
| **Super Admin** | Ward only. |

## Engagements

| Term | Definition |
| --- | --- |
| **Ladder Lift** | Internal concept for a Drawbackwards engagement that bridges sub-3.0 free scores into a Team subscription. Not yet productized. Pricing and packaging stay in memory. |
| **MSA** | Master Service Agreement. The standing contract with a Team customer. |
| **SOW** | Statement of Work. The specific scope under an MSA. |
| **The pool** | The shared score allowance on a Team or Pulse engagement (25,000 pooled on Team, custom on Pulse). |

## Don't say

| Avoid | Use instead | Why |
| --- | --- | --- |
| "AI-powered" | "Agent-native" or "scoring engine" | Positioning: infrastructure, not AI-feature |
| "Best-in-class" | A concrete claim | Buzzword |
| "Team plan" | "Teams" or "Teams engagement" | Teams isn't a plan, it's an engagement |
| "The Ladder platform" | "Ladder" | Platform is a generic word |
| "Tier" (for rungs) | "Rung" or "level" | "Tier" is the price tier |
| "Score it" (anonymous) | "Sign up to score" | No anonymous scoring |
